# 复选框 (Checkbox)

在用户需要从一组选项中选择零个、一个或多个选项，或者需要确认单一设置（如“同意服务条款”）时使用复选框。

## 如何识别复选框

复选框通常表现为：

- 一个方形框，选中时内部带有标记（通常是对号或半选的横线）。
- 始终与描述该选项内容的文本标签相邻显示。
- 在列表、表单或表格的左侧列中作为批量操作的选择开关。
- 允许在一组选项中进行多项选择的控件集合。

## 最佳实践

- **优先使用 life-ds 组件库中的复选框组件。**
- 复选框用于**多选**；如果选项是互斥的（只能选一个），请使用单选框（Radio）。
- 必须将复选框与其对应的文本标签关联起来（通常使用 `<label>`），确保点击文本也能触发复选框，从而扩大点击区域（Hit area）。
- 列表排列时，优先选择垂直排列以便于扫描和阅读。如果必须使用水平排列，请确保选项之间有足够的间距，避免用户将标签与错误的复选框混淆。
- 标签文本应清晰直接，通常使用肯定句，避免使用双重否定（如不要使用“不接收通知”并要求勾选）。
- 对于存在父子级关系的选项组（如全选/全不选），应正确实现“半选/不确定（Indeterminate）”状态。

## 常见布局

- **垂直选项组**：最常见且最易于阅读的布局，适合 3 个及以上的选项。
- **水平选项组**：仅在空间受限、选项较少且标签极短时使用。
- **表格行选择器**：位于表格的首列，表头通常包含一个控制当前页所有行的主复选框（支持全选和半选状态），单元格内通常只有一个checkbox而不需要右侧有文案。
- **独立确认操作**：例如表单底部的“我同意服务条款”，作为一个单独的验证步骤出现。

## 复选框尺寸规范

根据设计规范，复选框组件分为三种型号大小。在使用组件库复选框时，除了确保样式的正确引入，**必须**保证 HTML 结构的语义化和可用性：

| 尺寸名称            | 适用场景                       |
| :-------------- | :------------------------- |
| **大号 (Large)**  | 空间充裕、需要突出选择的操作，或作为整页的核心配置项 |
| **中号 (Medium)** | 绝大部分常规表单、列表或基础表格行选择器（默认选项） |
| **小号 (Small)**  | 空间极度受限的地方，如紧凑型表格、下拉菜单内嵌多选等 |

**示例代码**：

```tsx
import React from 'react';
import { Checkbox } from '@life-ds/components-web';

/**
 * ✅ 正确：使用 @life-ds/components-web 的 React 组件，不要手写 .lds-xxx 类名拼结构
 *
 * 约束：
 * - 右侧文案默认不显示；只有 showLabel={true} 才会显示 label
 * - 半选(indeterminate)通常是“父级部分选中”的表现，一般配合 readOnly 使用，避免点击把它当成普通 checked 切换
 * - 表格行选择器通常不需要右侧文案：直接 <Checkbox size="small" />
 */
export function CheckboxExamples() {
  return (
    <div style={{ display: 'grid', gap: 12 }}>
      {/* 1) 无文案（默认） */}
      <Checkbox size="default-size" />

      {/* 2) 有文案（显式开启 showLabel） */}
      <Checkbox size="default-size" showLabel label="多选框" />

      {/* 3) 默认选中（非受控初始值） */}
      <Checkbox size="default-size" showLabel label="已选中" defaultChecked />

      {/* 4) 半选（不确定态，推荐只读） */}
      <Checkbox size="default-size" showLabel label="半选" indeterminate readOnly />

      {/* 5) 禁用 */}
      <Checkbox size="default-size" showLabel label="禁用" disabled />
      <Checkbox size="default-size" showLabel label="禁用且选中" disabled defaultChecked />

      {/* 6) 表格行选择器（通常无文案 + 更紧凑） */}
      <Checkbox size="small" />
    </div>
  );
}

/**
 * ❌ 错误：不要直接写 HTML + className 拼装复选框结构
 * - 容易遗漏状态/无障碍属性
 * - 违反组件库的“必须用 React 组件”规范
 */
```

## 实现要点

在实现复选框时，请确保代码严格涵盖设计中定义的**所有 7 种核心交互状态**：

- **默认** (Default / 未选中)
- **悬停** (Hover - 鼠标移入时的边框高亮反馈)
- **选中** (Checked - 蓝色背景与白色对勾)
- **选中-禁用** (Checked Disabled - 浅蓝色背景与白色对勾，不可交互)
- **禁用** (Disabled / 未选中-禁用 - 灰色边框与浅灰背景，不可交互)
- **半选** (Indeterminate - 蓝色背景与白色横线，用于父级节点部分选中的状态)
- **半选-禁用** (Indeterminate Disabled - 浅蓝色背景与白色横线，不可交互)

> **注意**：除视觉状态外，必须有清晰的键盘聚焦（Focus）状态以支持无障碍访问。如果无法直接使用标准复选框，请使用仓库中现有的最接近的复选框组件，并保持层级、间距和状态行为的一致性。

## 兜底策略

如果模式不明确：

- 选择符合标准直觉的复选框变体（方形框），**绝对不要**将其设计得像开关（Switch）或单选框（Radio）。
- 保留原始界面的意图。
- 除非流程明确要求，否则避免发明自定义的复选框交互行为。
