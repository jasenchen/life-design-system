# 移动端页面规范

本规范适用于移动端 H5 / React 页面，不适用于 PC 后台或 Taro 小程序。

## 页面骨架

移动端页面由以下区域按需组成：

```text
移动端页面
├── 顶部导航区：返回、标题、少量辅助操作
├── 可滚动内容区：页面主体、列表、表单或详情
├── 浮层区：提示、弹层、底部 Sheet
└── 固定底部操作区：唯一主操作或少量高优先级操作
```

- 页面只保留当前任务所需的区域，不复用 PC 的顶部导航 + 侧边栏 + 内容画布框架。
- 可滚动内容区必须为固定底部操作区预留空间，避免最后一项被遮挡。
- 顶部导航、底部操作区和弹层需要考虑设备安全区；页面主体不应依赖固定的桌面宽度。
- 键盘弹出时，输入焦点与提交操作必须保持可见。

## 内容编排

- 一屏只建立一个清晰的主要任务和主要操作，次级操作收敛到文本操作、更多菜单或详情页。
- 列表优先使用可点击的 Cell / 信息行表达，避免在手机端直接呈现宽表。
- 筛选、日期、时间和多选操作优先采用适合触控的选择面板或底部 Sheet，不照搬 PC 下拉筛选栏。
- 表单按任务分组，字段、错误提示与操作反馈在同一视觉流中，避免过宽的横向 label 布局。
- 详情页使用信息分组与状态摘要组织内容，长内容分段展示，避免密集键值表。

## 固定区域与弹层

- 固定底部操作区仅放置当前流程必须保留的操作，并在滚动内容末尾预留对应空间。
- 全局短提示用于轻量反馈；需要用户确认、选择或完成复杂操作时使用 Dialog 或底部 Sheet。
- 弹层打开时必须锁定背景滚动，不使用背景模糊，仅保留压黑遮罩。
- 高层浮层使用 `bg-tips`，弹层表面使用 `bg-dialog`；不要用普通页面背景替代。

## 响应式与状态

- 以窄屏内容流为默认，不通过压缩 PC 栅格、侧栏或表格来实现移动端。
- 处理加载、空、错误、禁用与提交中状态；触屏操作不得依赖 hover 才能被理解。
- 页面不应出现无意义的横向滚动；确有横向内容时，必须明确其滚动边界和操作反馈。
