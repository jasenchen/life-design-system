# 表格 (Table)

在用户需要查看、比较、排序和操作大量结构化数据或同行同列的多维信息时使用表格。表格是一个容器模式，用于将相关的数据行组织成易于扫描的网格结构。

## 如何识别表格

可能的表格通常表现为：

- 由行和列组成的网格结构，顶部有一行明确的表头（列名）。
- 左侧可能包含用于批量操作的复选框列，右侧通常包含固定的操作列。
- 数据按字段严格对齐（如文本左对齐，数字/金额右对齐）。
- 底部通常附带分页器（Pagination）。
- 一致的行高节奏感，使许多相关的数据行感觉像是一个结构化的整体。

## 结构解析

将表格视为由几个稳定部分组成的结构：

- 表格容器（table container）：包含整个网格和可能的分页器的外层结构。
- 表头（table header）：定义每一列数据属性的顶部行，包含标题、对齐方式、排序或过滤控件。
- 表身/数据行（table body / data rows）：按照相同节奏对齐的重复单元格组合，承载实际数据。
- 单元格（table cell）：承载具体数据、状态标签或操作按钮的最小单位。
- 分隔处理（divider treatment）：用于分隔行和列的线条、背景斑马纹或间距。
- 底部区域（footer）：包含分页控件或数据汇总统计的区域。

## 表头与表身组件规范

根据设计规范，表格内部的核心组件分为**表头**和**表身**两大部分，它们各自支持特定的内容类型和交互状态：

### 1. 表头 (Table Header)

表头用于定义列的属性和全局操作，主要支持以下特征与状态：

- **对齐方式**：支持 左对齐、中对齐、右对齐。**原则上，表头列名的对齐方式必须与该列表身内容的对齐方式保持完全一致。**
- **功能组件**：
  - **列名文本**：标准表头标题。
  - **排序/筛选**：标题旁附带的上下排序箭头或漏斗筛选图标。
  - **多选**：通常位于首列，用于全选当前页的所有表身数据。
- **交互状态**：包含 **正常 (Default)**、**悬停 (Hover)**（背景色轻微加深）、**点击/激活 (Active)** 三种视觉状态。

### 2. 表身 (Table Body)

表身用于承载具体的数据，每行/每个单元格支持 **正常 (Normal)** 和 **悬停 (Hover，整行背景色加深以辅助视觉定位)** 两种核心交互状态。表身单元格支持极高丰富度的组件类型，且具有明确的**默认对齐规范**：

- **基础数据与文本**：
  - **文本**（左对齐）：常规单行文本。
  - **双排文本**（左对齐）：主标题 + 灰色副标题/ID 的上下堆叠文本。
  - **金额**（右对齐）：如 ¥240,000.00，数字类通常右对齐以便比较。
  - **日期/时间**（左对齐）：标准化的时间格式展示。
- **状态与控件**：
  - **标签 (Tag)**（中对齐）：用于展示状态（如成功、失败、进行中）。
  - **开关 (Switch)**（中对齐）：用于直接在表格内控制某项功能的启停。
  - **操作 (Action)**（左对齐）：行内的文字按钮组合（如“查看 编辑 更多”）。
- **复合图文结构（统一左对齐）**：
  - **图片 + 双排文本**
  - **头像 + 双排文本**
  - **图片 + 三排文本**：主标题 + 标签行 + 辅助说明行。
  - **头像 + 三排文本**
  - **视频 + 三排文本**
- **行级控制组件 (通常位于首列或特定列，居中对齐)**：
  - **多选**：单行的复选框。
  - **折叠/展开**：用于触发展开行表格的箭头（`>`）。
  - **排序**：用于手动调整行顺序的拖拽手柄（`⋮⋮`）。

> **对齐原则总结**：除了金额/数字明确要求**右对齐**，以及行级控制组件（多选、折叠、排序）要求**居中对齐**外，其余绝大部分文本、图文复合结构、状态标签及操作按钮均默认**左对齐**。

## 最佳实践

- **优先使用 life-ds 组件库中的表格（Table）组件。**
- 当需要一起扫描、比较或管理几行相关的数据时，请使用表格。如果是移动端或卡片式展示，请考虑使用列表（List）。
- **对齐方式**：文本类型数据左对齐，数字或金额类型右对齐，状态标签或简短固定操作可居中对齐。
- **空白数据**：单元格没有数据时，应显示破折号（如 `-` 或 `--`），而不是留白或显示“null”，以明确告知用户该数据缺失而非未加载。
- **操作收纳**：当单行操作超过 3 个时，应将前两个最高频的操作平铺展示，其余操作收纳进“更多（...）”下拉菜单中。
- **列宽控制**：为重要列设定合理的最小宽度，对超长文本使用省略号截断（配合 Tooltip 提示），防止破坏表格整体布局。
- **冻结行列**：当列数过多需要横向滚动时，必须冻结最左侧的关键信息列（如 ID、名称、多选框）和最右侧的操作列。表头也应在向下滚动时吸顶冻结。
- 让表格定义整体的结构和对齐方式，让单元格定义具体的数据展示和行内交互。

## 常见布局

- **基础数据表**：标准的行列展示，配合底部分页器，适用于大部分查询展示场景。
- **带批量操作的表格**：表格左侧首列包含复选框，表格上方（通常与筛选器同行或在表格顶部工具栏）提供批量删除、导出等按钮。
- **展开行表格（Expandable）**：当单行数据需要展示详细的附属信息（如子任务、长描述）时，提供展开/收起图标。
- **树形表格（Tree Table）**：用于展示具有层级关系（父子级）的数据结构。

## 开箱即用单元格 (Cells)

表格内部有一些开箱即用的 “Cell 组合组件”，用于快速拼出常见的表格内容结构。AI 生成代码时应优先使用这些组件，而不是在 `Td` 里自行拼接复杂 DOM。

| Cell 组件              | 典型场景                                | 关键 props                                                                                        | 输出/注意事项                                                                                             |
| :------------------- | :---------------------------------- | :---------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------- |
| `TableCellProduct`   | 商品信息列：图片 + 标题 +（可选）标签 + 商品 ID（双排信息） | `img`（图片 URL）、`title`（主标题）、`id`（商品 ID）、`tag?`（可选标签文本）、`tagVariant?`（`default \| orange \| red`） | 内部已包含一个 `Tag` 用于显示 `tag`，并把 `tagVariant` 映射为 `Tag` 的 `variant/color`；如果你的产品信息结构不同（例如三排文本/头像），再考虑自定义 |
| `TableCellAmount`    | 金额/价格列：用于展示价格、成交额等数字数据              | `children`（金额文本）                                                                                | 使用基础正文字号与数字字体族 `--font-number`；对齐方式仍由表格列的布局规则决定                                                               |
| `TableCellOperation` | 操作列：行内多个操作聚合（如“上架 / 编辑 / 更多”）       | `children`（一组操作项）                                                                               | 仅负责操作项的横向排列与间距；一般与 `TableCellAction` 组合使用                                                           |
| `TableCellAction`    | 具体操作项：行内文字操作（看起来像链接）                | 继承 `<a>` 的 props（如 `href` / `onClick`），以及 `danger?`（危险操作）                                       | 渲染为 `<a>`；危险操作用 `danger` 强调（如删除/下架）                                                                 |

## 实现要点

在实现表格时，请确保代码涵盖以下状态和功能：

- 数据加载中 (Loading State - 通常使用骨架屏或半透明遮罩加 Spinner)
- 空状态 (Empty State - 当没有数据或查询无结果时，展示友好的空状态插画和文案)
- 悬停高亮 (Row Hover - 鼠标移入单行时整行背景色加深，帮助视觉定位)
- 排序状态 (Sortable - 升序、降序及取消排序的图标高亮状态)
- 选中状态 (Selected Row - 勾选复选框后整行高亮)
- 滚动阴影 (Scroll Shadow - 冻结列在滚动时产生的边缘阴影分隔)
- 分页状态 (Pagination - 当数据量大时正确展示分页器，并处理翻页逻辑)

如果无法直接使用标准表格，请使用仓库中现有的最接近的表格组件，并保持对齐、间距、边框颜色和交互行为的一致性。

## 兜底策略

如果模式不明确：

- 选择最基础的默认尺寸、无冻结列的标准网格表格。
- 保证表头清晰，数据左对齐。
- 当未解决的问题是关于单条数据的设计，且不需要严格的列对齐时，考虑切换到列表（List）模式。
- 除非数据结构极其特殊（如高度定制的甘特图或日历视图），否则避免用 DIV/CSS 自己强行拼凑复杂的表格布局。
